2. **Double Message Submission:** When confirming a revert, the reverted prompt text populated the desktop text editor. However, when the user subsequently sent that message from their phone, it resulted in the text being concatenated and sent twice (e.g. `reverted prompttextreverted prompttext`).

---

## 🎭 ACT II: Diagnosing the Remaining Copy and Double Send Issues
We investigated the codebase and identified the root causes:
1. **Layout-Dependent `innerText` on Cloned Nodes:** In mobile browsers (Chrome and Safari), calling `.innerText` on a cloned, disconnected DOM node (which is not attached to the layout/render tree of the document) returns an empty string. The copy button failed because it was querying `.innerText` on a cloned article element, yielding empty text.
2. **Out-of-Sync React State on Deletion:** In `server.js`'s `injectMessage()`, the server cleared the desktop editor using separate `selectAll` and `delete` commands before inserting the new text. In modern React-controlled rich text editors (like Lexical/ProseMirror), these separate mutations did not trigger the intermediate deleted state synchronization properly, causing the subsequent `insertText` to append the text instead of overwriting it, resulting in duplication.

---

## 🎭 ACT III: Implementing the Final Fixes
We applied targeted edits to `public/js/app.js` and `server.js`:
1. **Targeted Agent Copy (app.js):** Updated the agent copy handler to query the specific text content container (`.select-text` or `.px-2.py-1`) of the ancestor article directly. It clones only that container and prunes style blocks, buttons, and SVGs, avoiding layout-dependency issues and providing fallback options.
2. **Editor Match Bypass & Overwrite (server.js):** Updated `injectMessage()` to compare the desktop text and bypass insertion entirely if the text is identical, preventing any state duplication. If the text differs, it uses atomic selection replacement (`selectAllChildren` for contenteditables and `select()` for textareas) to replace the selection in a single atomic overwrite event, which React registers and synchronizes correctly.
