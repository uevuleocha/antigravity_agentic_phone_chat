## Context

Phone Connect was built targeting the Antigravity IDE (VS Code-based), whose DOM uses:
- `#conversation`, `#chat`, `#cascade` — chat container IDs
- `lucide-clock`, `lucide-history` SVG classes — history button detection
- `'Fast'` / `'Planning'` leaf text nodes — mode toggle detection

The Antigravity 2.0 **agentic desktop app** uses a completely different React-based DOM:
- `data-testid="conversation-view"` wraps the chat area
- `.scrollbar-hide.overflow-y-auto` (scrollH: 4418) is the live chat scroll container
- `data-testid="convo-pill-<UUID>"` spans enumerate conversation history directly in the sidebar
- A `DIV` with text `"Conversation History"` and `cursor: pointer` is the history nav element
- **No Fast/Planning mode UI exists at all** — the agentic app does not have this concept

Live DOM inspection via CDP was completed on 2026-06-14. All selectors below are verified from the live running app.

### Phase H — Dynamic Target Switching & Race Condition Fix — ✅ Complete

**Goal:** Correctly sync to the active/focused Electron page target, and ensure switched conversation text renders and stays visible on the phone.

**Root cause:** 
1. The server binds permanently to the first page target discovered at startup. When the user switches chats, the server remains connected to the background target, causing a blank screen because Electron does not layout/render background pages.
2. In `app.js`'s `loadSnapshot()`, style overrides only target `#conversation, #chat, #cascade`, neglecting the agentic `[data-testid="conversation-view"]` and `.scrollbar-hide` elements, causing them to collapse or overflow invisibly.
3. A client-side race condition exists where `checkChatStatus()` calls `showEmptyState()` mid-transition and wipes out fresh snapshots.

**Tasks:**

*Server-side (server.js — `initCDP()`, `ensureActiveCDP()`, `startPolling()`, `/select-chat` route):*
- [x] In `initCDP()`, assign `cdpConnection.port` and `cdpConnection.url`.
- [x] Implement `ensureActiveCDP()` focus detector: checks current target focus with `document.hasFocus()`, falls back to checking all page targets in `/json/list`, and swaps connection when a different page holds focus.
- [x] Call `await ensureActiveCDP()` at the beginning of each `startPolling()` cycle.
- [x] In `/select-chat` route, check `/json/list` for a page target with the matched title, and immediately swap connection to it.
- [x] Keep a delay and snapshot trigger inside `selectChat()` to ensure DOM finishes rendering before returning.
- [x] Add `// [Agentic App Fix - Phase H]` comments.

*Client-side (app.js — `loadSnapshot()`, `selectChat()`, `checkChatStatus()`):*
- [x] Add `[data-testid="conversation-view"]` and `.scrollbar-hide` to the styling overrides in `loadSnapshot()` (lines 242, 252, 261, 271).
- [x] Set `isSwitchingConversation = true` when starting `selectChat()`, suppress `showEmptyState()` in `checkChatStatus()` while true, and clear after success or an 8-second safety timeout.
- [x] Cancel the polling loop in `selectChat()` early if `loadSnapshot()` loads valid elements.

**Verification:**
- [x] Switch chats from phone → text renders and stays visible.
- [x] Switching chats on the desktop app triggers the server to connect to the new active page.
- [x] Stats (nodes and KB) update correctly per-chat.
