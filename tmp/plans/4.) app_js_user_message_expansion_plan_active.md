# PhoneConnect - Expand long user messages when clicked

Status: ACTIVE / VERIFIED COMPLETE - created 2026-06-15.

Plan lifecycle: ACTIVE on 2026-06-15. This plan covers implementing click delegation and CDP remote click routing to allow phone users to expand visually compacted long user messages.

---

## Context

In the React-based Antigravity 2.0 Agentic desktop app, user messages that are long are visually compacted into a box with a maximum height of 10vh and a linear gradient mask. The wrapper element of this compacted text contains the `cursor-pointer` class. Clicking this element toggles React state to expand/collapse the message text.

On the phone connect client, the static HTML snapshot is injected into the DOM, stripping all React event handlers. Tapping on a compacted user message bubble does nothing because no event listener is attached on the mobile frontend to forward the click.

By intercepting taps on these specific elements in `public/js/app.js` and forwarding them via POST to `/remote-click` with the selector and calculated index, we can simulate the click on the desktop app, which will toggle the state and update the phone's view on the next snapshot.

## Phased Implementation Plan

### Phase A - Phone UI: Add Click Delegation in `public/js/app.js`
- **Goal:** Intercept clicks on compacted user messages and send them to the server.
- **Tasks:**
  - Locate the `chatContainer` click event listener in `public/js/app.js`.
  - Add a check for `e.target.closest('[data-testid="user-input-step"] .cursor-pointer')`.
  - If matched, check that the click is not on a child button (like Copy or Revert) to prevent interference.
  - Calculate the index of the clicked element relative to all elements matching `[data-testid="user-input-step"] .cursor-pointer` inside `chatContainer`.
  - POST to `/remote-click` with the selector `[data-testid="user-input-step"] .cursor-pointer` and the calculated index.
  - Call `setTimeout(loadSnapshot, ...)` to trigger a rapid reload.

### Phase B - Verification and testing
- **Goal:** Ensure the feature works correctly without regressions.
- **Tasks:**
  - Verify that clicking a collapsed long user message expands it.
  - Verify that clicking copy/revert buttons on user messages still works correctly.
  - Verify that short user messages (which lack the `.cursor-pointer` class) do not trigger any remote clicks.
