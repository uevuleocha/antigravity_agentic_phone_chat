# Antigravity Phone Connect — Message Action Buttons Support

Status: ACTIVE / PENDING - created 2026-06-15.

Plan lifecycle: ACTIVE on 2026-06-15. This plan covers implementing client-side copy actions and CDP-forwarded undo and thumbs up/down actions for message bubbles in the phone interface.

---

## Context

In the Antigravity 2.0 Agentic Desktop App, every message bubble contains interactive action buttons:
- For user messages: a Copy button (`aria-label="Copy"`) and a Revert/Undo button (`data-testid="revert-button"`).
- For agent responses: a Copy button (`aria-label="Copy"`), a Thumbs Up button (`aria-label="Good response"`), and a Thumbs Down button (`aria-label="Bad response"`).

When Phone Connect captures a DOM snapshot, these buttons are cloned and served to the phone. However, because DOM event listeners are not cloned, and because the Phone Connect UI client has no handlers for them, these buttons are currently inert on the phone.

To make them work:
1. **Copy button:** Must copy the message text locally to the phone's native clipboard (not the desktop's).
2. **Undo/Feedback buttons:** Must calculate their DOM selectors/indices and forward the click to the desktop app via CDP using the `/remote-click` route.

---

## Locked Decisions

| # | Decision | Rationale |
|---|---|---|
| 1 | Modify only `public/js/app.js` | Keeps changes purely client-side; keeps `server.js` clean and preserves upstream merge paths. |

| 3 | Use existing `/remote-click` endpoint | The existing endpoint in `server.js` is fully generic and handles element focusing and clicking via CDP. |

## Prompt Contract

```markdown
GOAL:
  (1) Clicking the copy button on an agent or user message on the phone copies the full message text locally to the phone's clipboard.
  (2) Clicking the undo arrow (revert) on a user message on the phone rolls back the desktop app conversation and updates the phone screen.
  (3) Clicking thumbs up/down on an agent message on the phone clicks the corresponding feedback button on desktop.

CONSTRAINTS:
  - All changes must be client-side only and confined to public/js/app.js.
  - No edits to server.js; use the existing generic /remote-click endpoint.
  - Clipboard copy must occur locally on the phone browser.
  - All new code must be marked with // [Agentic App Fix - PhaseX] comments.
  - Existing IDE fallbacks and code block copy buttons must not be degraded.

DEFINITION OF DONE:
  - Copying user/agent messages works on the phone, copying the exact text without UI element names or metadata.
  - Tapping the undo button on an earlier user message on the phone triggers a revert on desktop and refreshes the phone UI.
  - Tapping thumbs-up/down on the phone updates the feedback button state on the desktop app.
  - All changes are committed and pushed to the origin repository on GitHub.

FAILURE (any of these = not done):
  - Clicking copy on the phone copies nothing or copies the desktop computer's clipboard instead.
  - Code block copy buttons in pre tags are broken or fail to copy code.
  - Modifications are made to server.js.
```

## Phased Implementation Plan

### Phase A - Implement Client-Side Copy Action for Message Bubbles - [Status: Implemented / Debug Pending]
- **Goal:** Enable local copying of message text on the phone client.
- **Tasks:**
  - In `public/js/app.js`, add a click listener under the chat container to intercept buttons with `aria-label="Copy"`.
  - Ensure we exclude code block copy buttons (which use `aria-label="Copy code"`).
  - Find the parent message container: `.closest('[data-testid="user-input-step"]')` for user messages, or `.closest('[role="article"]')` for agent responses.
  - Extract the text content: query for `.whitespace-pre-wrap` (user) or `.select-text` (agent).
  - Call the local `copyToClipboard(text)` function.
  - Dim or flash the copy button to provide brief visual feedback.
- **Verification:**
  - Click copy on a user message on the phone and paste it to verify the clipboard text.
  - Click copy on an agent response on the phone and paste it to verify the clipboard text.

---

### Phase B - Implement Revert/Undo Action Forwarding - [Status: Implemented / Debug Pending]
- **Goal:** Enable conversation rollbacks from the phone interface.
- **Tasks:**
  - Add click delegation in `public/js/app.js` for buttons with `data-testid="revert-button"` or `aria-label="Undo changes up to this point"`.
  - Calculate the element's index among all active undo buttons in the `chatContainer` DOM.
  - POST to `/remote-click` with the selector `'button[data-testid="revert-button"]'` and the index.
  - Trigger `loadSnapshot()` after a short delay (e.g., 500ms and 1500ms) to reload the updated conversation.
- **Verification:**
  - Click the undo arrow on a user message, check if the desktop app reverts the conversation, and check if the phone updates to show the reverted state.

---

### Phase C - Implement Thumbs Up/Down Forwarding - [Status: Implemented / Debug Pending]
- **Goal:** Enable rating agent responses from the phone.
- **Tasks:**
  - Add click delegation for buttons with `aria-label="Good response"` and `aria-label="Bad response"`.
  - Calculate their indices and POST to `/remote-click` with the appropriate selector.
- **Verification:**
  - Click thumbs up/down on the phone, and verify the rating highlights on the desktop screen.

---

- **Tasks:**
  - Run static check `node --check server.js`.
  - Commit changes with message `feat: message bubble action buttons support`.
  - Push to origin master branch.
- **Verification:**
