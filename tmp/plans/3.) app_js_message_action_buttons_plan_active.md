# Antigravity Phone Connect — Message Action Buttons Support

Status: ACTIVE / PENDING - created 2026-06-15, updated 2026-06-15.

Plan lifecycle: ACTIVE on 2026-06-15. This plan covers implementing copy actions, CDP-forwarded undo and rating actions, full-screen confirm-undo modal forwarding, and revert input text box value syncing.

---

## Context

In the Antigravity 2.0 Agentic Desktop App, every message bubble contains interactive action buttons:
- For user messages: a Copy button (`aria-label="Copy"`) and a Revert/Undo button (`data-testid="revert-button"`).
- For agent responses: a Copy button (`aria-label="Copy"`), a Thumbs Up button (`aria-label="Good response"`), and a Thumbs Down button (`aria-label="Bad response"`).

When Phone Connect captures a DOM snapshot, these buttons are cloned and served to the phone. However, because DOM event listeners are not cloned, and because the Phone Connect UI client has no handlers for them, these buttons are currently inert on the phone.

During initial implementation:
- User-side copy button worked.
- Revert/Undo click successfully triggered the revert popup on desktop, but:
  1. The "Confirm Undo" modal is rendered outside `conversation-view` in a React portal. It was invisible on the phone client, blocking the session.
  2. Confirming the undo on desktop copies the reverted message to the desktop input area, but this text box state was not synced to the phone client's input box.
- Agent-side copy button and ratings buttons failed to respond.

To make them work:
1. **Copy button:** Must copy the message text locally to the phone's native clipboard (not the desktop's). Agent copy must strip the feedback and button panel elements reliably.
2. **Undo/Feedback buttons:** Must calculate their DOM selectors/indices and forward the click to the desktop app via CDP using the `/remote-click` route.
3. **Confirm-Undo Modal:** `captureSnapshot()` must scan `document.body` for overlay dialogs and append them to the snapshot so they render as overlays on the phone.
4. **Revert Text Box Sync:** `captureSnapshot()` must query the desktop text editor's value. The phone client will sync this value to its native input area upon confirming a revert.

## Prompt Contract

```markdown
GOAL:
  (1) Clicking the copy button on an agent or user message on the phone copies the full message text locally to the phone's clipboard.
  (2) Clicking the undo arrow (revert) on a user message on the phone rolls back the desktop app conversation, forwards the Confirm/Cancel modal to the phone, and syncs the reverted input text to the phone.
  (3) Clicking thumbs up/down on an agent message on the phone clicks the corresponding feedback button on desktop.

CONSTRAINTS:
  - Edits are restricted to public/js/app.js and server.js.
  - Clipboard copy must occur locally on the phone browser.
  - All new code must be marked with // [Agentic App Fix - PhaseX] comments.
  - Existing IDE fallbacks and code block copy buttons must not be degraded.

DEFINITION OF DONE:
  - Copying user/agent messages works on the phone, copying the exact text without UI element names or metadata.
  - Tapping the undo button on an earlier user message on the phone triggers the revert modal on both desktop and phone.
  - Confirming the undo on the phone rolls back history and populates the phone's message input box with the reverted prompt.
  - Tapping thumbs-up/down on the phone updates the feedback button state on the desktop app.
  - All changes are committed and pushed to the origin repository on GitHub.

FAILURE (any of these = not done):
  - Clicking copy on the phone copies nothing or copies the desktop computer's clipboard instead.
  - The "Confirm Undo" modal is not displayed on the phone, causing a hang.
  - Code block copy buttons in pre tags are broken or fail to copy code.
```

### Phase E - Fix Agent's Side Copy & Rating Actions - [Status: Complete]
- **Goal:** Guarantee agent copy and rating click forwarding work reliably on all bubbles.
- **Tasks:**
  - Rewrite agent copy text extraction to find the message text container (`.select-text` or `.px-2.py-1`) directly. Clone only this container and remove `<style>`, `<svg>`, and `<button>` tags, bypassing layout-dependent `innerText` empty-string bugs on disconnected cloned nodes in mobile browsers.
  - Add console logs and fallback checks to guarantee copying succeeds on all platforms.
- **Verification:**
  - Click agent copy button on phone, verify the copied response matches plain text.
  - Click thumbs up/down on agent message on phone, check desktop highlighting.

---

### Phase F - Forward Desktop Modal Dialogs to Phone - [Status: Complete]
- **Goal:** Allow modal dialogs (like "Confirm Undo") to be visible and interactive on the phone.
- **Tasks:**
  - In `server.js`'s `captureSnapshot()`, query for any active modal using `.animate-modalFadeIn, [role="dialog"], .bg-overlay-subtle`.
  - If a modal exists, clone it, strip `data-ag-rem` attributes, and append it to the cloned chat container.
  - In `public/js/app.js`, add a dedicated modal click-delegator that captures clicks inside modal overlays, calculates indices within the modal container, and forwards clicks to `/remote-click` with correct index and no text content filtering. This makes Confirm, Cancel, and the top-right X button work.
- **Verification:**
  - Click undo arrow on phone, check if the "Confirm Undo" modal overlays the phone screen.
  - Click "Cancel" on the phone, check if the modal dismisses on both desktop and phone.

---

### Phase G - Sync Revert Input Value to Phone - [Status: Complete]
- **Goal:** Auto-populate the phone's message input box when an undo action is confirmed and prevent duplicate sends.
- **Tasks:**
  - In `server.js`'s `captureSnapshot()`, query the active desktop text editor's value and include it as `inputValue` in the snapshot JSON.
  - In `public/js/app.js`, check if the modal is open using `chatContainer.querySelector(...)`. Prevent the `isAwaitingRevertInput` sync logic from executing or clearing while the modal is open.
  - Once the modal is closed and `isAwaitingRevertInput` is true, sync `inputValue` to `messageInput.value`, resize the text input to fit the content, and clear the flag.
  - Update `injectMessage()` in `server.js` to compare the desktop text and bypass insertion on match, or use single atomic selection overrides (`selectAllChildren` / `select()`) to resolve out-of-sync React text duplication.
- **Verification:**
  - Click undo arrow on phone, tap "Confirm", verify that the reverted prompt is automatically pasted and sized in the phone's chat input area.

- **Verification:**

### Phase J - Precedence-First Message Injection (server.js) - [Status: Pending]
- **Goal:** Ensure message edits/resends from the phone take precedence and sync correctly with the desktop editor state (Lexical/React).
- **Tasks:**

  - Execute `document.execCommand('insertText', false, textToInsert)` to replace selection atomically, updating Lexical's internal state.
  - Trigger `input` and `change` events on the editor to force state synchronization.
- **Verification:**
  - Edit or resend a message from the phone, verify that it successfully overwrites the desktop editor's text and is submitted instead of ignored.
